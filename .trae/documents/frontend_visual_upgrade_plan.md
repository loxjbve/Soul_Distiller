# 前端视觉全面升级与精简计划

## 1. 意图与目标
- **核心目标**：全方位升级前端页面的视觉体验，转向“毛玻璃与流光 (Glassmorphism)”风格，同时对冗余信息和复杂的看板进行减法，聚焦核心工作流。
- **关键操作**：
  1. **简化文档看板**：将项目详情页中繁重的四列 Kanban 看板改为更紧凑、直观的统一网格列表 (Grid Layout)。
  2. **精简首页信息**：去除首页 (`index.html`) 冗长的说明文字，精简头部区域，让项目列表成为视觉焦点。
  3. **增强视觉呈现**：全局采用毛玻璃 (`backdrop-filter`)、深色半透明面板、动态渐变边框和呼吸光晕，大幅提升页面的科技感和华丽度。

## 2. 涉及文件与详细修改点

### 2.1 `app/static/style.css` (毛玻璃、流光与网格样式)
- **动态背景与环境光**：
  - 增强现有的 `.page-aura`，调整色彩组合，让背景的流光更加柔和深邃。
  - 全局字体和排版微调，增大卡片间距。
- **面板升级 (Panels & Cards)**：
  - 更新 `.workspace-panel`, `.hero-board`, `.project-hero`：
    - 背景改为高透度的 `rgba(17, 24, 39, 0.25)`。
    - 增加强力的 `backdrop-filter: blur(24px)`。
    - 加入伪元素实现的“流光边框动画” (`::before` 线性渐变扫描效果)。
- **清理冗余代码**：删除所有 `.kanban-board`, `.kanban-column`, `.kanban-column-header` 相关的 CSS。
- **重塑文档卡片 (`.doc-card`)**：
  - 强化 `.doc-card` 的毛玻璃质感，添加精致的内部阴影 (`box-shadow: inset 0 1px 1px rgba(255,255,255,0.1)`) 和悬停时的光效放大。
  - 优化 `.document-grid` 布局参数，确保适配不同数量的文档。

### 2.2 `app/templates/index.html` (首页精简)
- **Hero 区域做减法**：
  - 移除冗长的 `<p class="hero-note">` 段落。
  - H1 标题精简为更具科技感和力量感的短句（如：`构建可交互的人物 Skill。`）。
  - 保留底部的环境检测 Chips（大模型和向量模型状态），并用流光边框包裹。
- **项目列表增强**：
  - 让 `.project-row` 采用与整体一致的毛玻璃和悬浮光晕设计，突出“开始新项目”和“已有项目”的核心入口。

### 2.3 `app/templates/project_detail.html` (详情页看板重构)
- **重构 HTML 结构**：
  - 删除包含 4 列看板的 `<div class="kanban-board">...</div>` 结构。
  - 替换为单一的 `<div class="document-grid" id="document-grid"></div>`。
  - 保留并美化现有的 `.status-stack`（位于侧边栏，作为全局统计）。
- **重构 JS 渲染逻辑 (`renderDocuments`)**：
  - 废除将文档分配到 4 个数组（`unprocessed`, `queued`, `processing`, `completed`）的逻辑。
  - 直接生成统一的文档卡片 HTML，并整体插入 `#document-grid`。
  - **排序逻辑**：按照优先级排序（如：失败 > 进行中 > 队列中 > 未处理 > 已完成），确保用户最关心的“正在处理”和“异常”文件始终位于网格前列。
  - 删除多余的 DOM 更新调用（如 `count-unprocessed` 等），依赖侧边栏统计即可。

## 3. 验证步骤
1. **视觉效果检查**：加载页面，确认背景的流光动画、所有面板的毛玻璃模糊效果 (`backdrop-filter`) 和发光边框正常渲染。
2. **首页精简检查**：确认首页说明文字已被精简，视觉焦点集中在项目列表上。
3. **详情页布局检查**：确认项目详情页的“四列看板”已消失，替换为整齐美观的文档网格，且原有的处理进度条（橙绿渐变）依然能在卡片中正常动态显示。
4. **功能连贯性测试**：上传新文档，观察文档是否直接在统一网格中出现，并且状态更新（解析中 -> 索引中 -> 完成）正常运作。排序机制是否将正在处理的文档置前。
